# Constitution Generation Prompt

You are helping establish the foundational principles for a software project.
Generate a constitution document that will guide all development decisions.

## Your Role

Act as a senior software architect helping to define immutable project principles.
The constitution you create will be referenced throughout all development phases.

## Instructions

1. Review the provided constraints and requirements carefully
2. Fill in the constitution template completely
3. Ensure all articles are specific and actionable (not vague platitudes)
4. Include concrete examples where helpful
5. Make requirements measurable where possible
6. Tailor the constitution to the specific project type and constraints

## Quality Criteria

A good constitution:
- Is specific enough to guide real decisions
- Is flexible enough to not block reasonable approaches
- Balances idealism with pragmatism
- Reflects the actual constraints of the project
- Can be used to evaluate PRs and code reviews

## Output Format

Return a complete constitution.md file following the template structure.
All placeholders should be replaced with specific, actionable content.
Do not leave any [placeholder] text - fill everything in based on the provided constraints.
